# Execution Board Template

## Session metadata

- Session ID
- Owner
- Dependencies
- Estimated effort
- Risk level

## Objective

The exact outcome this session must produce.

## Why now

Why this session must happen at this stage, and what risk it removes.

## Inputs

The approved files, artifacts, and prior outputs required before starting.

## Tasks

Write tasks with:

- primary tasks
- control subtasks
- required execution order

## Checks

Each check must include:

- pass/fail condition
- how to verify
- common failure signs

## Gate

List multiple gate conditions and mark which ones are blocking.

## Outputs

The exact artifacts or state changes this session must produce.

## Completion standard

State explicitly:

- what concrete artifact must exist before the session can be marked complete
- what does **not** count as completion
- whether approval is required in addition to artifact creation

## Fallback

Fallbacks must:

- track the main risks of the session
- state which fallback remains valid under governance and contest rules

## Not allowed

Must contain:

- relevant non-negotiables
- session-specific forbidden actions

## Approval needed

Must state:

- what needs approval
- who approves it
- how far the session can continue without that approval

## Hand-off to next session

Must state:

- hand-off outputs
- preconditions for the next session
- readiness state:
  - not ready
  - partial
  - ready
